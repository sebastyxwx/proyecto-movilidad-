Requerimientos Funcionales

1.0. Registro e inicio de sesión:
El sistema debe permitir a los usuarios registrarse con correo electrónico y contraseña, e iniciar sesión de forma segura.

2.0. Consulta de rutas:
El usuario podrá buscar rutas disponibles ingresando origen y destino.

3.0. Visualización de horarios:
El sistema mostrará horarios estimados de buses, taxis o transporte disponible.

4.0. Estado del tráfico en tiempo real: 
La plataforma mostrará información actualizada sobre congestión vehicular y accidentes.

5.0. Ubicación en mapa:
 El usuario podrá visualizar su ubicación actual y rutas recomendadas en un mapa interactivo.

6.0. Notificaciones:
 El sistema enviará alertas sobre retrasos, cierres viales o cambios de ruta.

7.0. Historial de viajes:
 El usuario podrá consultar viajes realizados anteriormente.

8.0. Calificación del servicio:
 Los usuarios podrán calificar rutas o servicios de transporte utilizados.


2.1. Requerimientos No Funcionales


1.1. Seguridad:
 La información personal de los usuarios debe estar protegida mediante cifrado de datos.

2.1. Disponibilidad:
La plataforma debe estar disponible las 24/7.

3.1. Rendimiento:
 Las consultas de rutas deben responder en menos de 3 segundos.

4.1. Usabilidad:
La interfaz debe ser sencilla, intuitiva y fácil de usar.

5.1. Compatibilidad:
El sistema debe funcionar en computadores, tabletas y teléfonos móviles.

6.1. Escalabilidad:
 La plataforma debe soportar un aumento de usuarios sin afectar el rendimiento.

7.1. Mantenimiento:
 El código debe estar documentado para facilitar futuras mejoras.
